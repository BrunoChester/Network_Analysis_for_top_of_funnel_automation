How last-touch attribution works

Last-touch attribution tracks the final interaction that led to opportunity creation. It follows a simple hierarchy:

Attribution Hierarchy (60-day window)
  • Partner: Referrals from integration partners or resellers
  • Customer Success: Expansion opportunities identified by CS team
  • Sales Outbound: Direct outreach activities (emails, calls, LinkedIn)
  • Marketing Touch: Events, content downloads, webinars, ads
  • Product: Trial sign-ups, in-product conversion flows

The key insight: not all pipeline is created equal. A $50K deal from a warm inbound lead has different implications than the same deal from cold outbound. The source tells you what to scale.

Calculations blueprint

Understanding the math behind attribution metrics ensures consistent measurement and accurate analysis. Here's how each metric in the dashboard is calculated:

Pipeline Source Metrics
  • Pipeline Amount: Sum of all opportunity amounts where last_touch_source = [source_type]
  • % of Total Pipeline: (Source Pipeline Amount Ã· Total Pipeline Amount) Ã— 100
  • % vs Goal: (Actual Source Pipeline Ã· Source Target) Ã— 100
  • Avg Deal Size: Source Pipeline Amount Ã· Number of Opportunities
  • Number of Oppt: COUNT(opportunities) WHERE last_touch_source = [source_type]
Quarter Performance Metrics
  • Generated: SUM(all opportunity amounts) for date range
  • Attainment: (Total Generated Ã· Quarter Target) Ã— 100
  • Opportunities: COUNT(all opportunities) for date range
  • Avg Deal Size: Total Generated Ã· Total Opportunities
Rep Performance Metrics
  • Source Pipeline by Rep: SUM(opportunity amounts) WHERE owner = [rep] AND last_touch_source = [source]
  • Total by Rep: SUM(all source pipeline amounts) for individual rep
  • Top Source: MAX(source pipeline amount) with percentage of rep total
  • Top Source %: (Highest Source Amount Ã· Rep Total) Ã— 100
# Sample SQL for pipeline source calculation with sales cycle SELECT last_touch_source, SUM(amount) as pipeline_amount, COUNT(*) as opportunity_count, AVG(amount) as avg_deal_size, SUM(amount) / (SELECT SUM(amount) FROM opportunities) * 100 as pct_total, AVG(DATEDIFF(close_date, created_date)) as avg_cycle_days FROM opportunities WHERE created_date >= '2024-10-01' AND created_date <= '2024-12-31' AND stage != 'Closed Lost' AND close_date IS NOT NULL # Only include closed deals for cycle calculation GROUP BY last_touch_source ORDER BY pipeline_amount DESC; # Alternative: Include all pipeline with separate cycle calculation SELECT o.last_touch_source, SUM(o.amount) as pipeline_amount, COUNT(o.*) as opportunity_count, AVG(o.amount) as avg_deal_size, AVG(CASE WHEN o.stage = 'Closed Won' THEN DATEDIFF(o.close_date, o.created_date) ELSE NULL END) as avg_cycle_days_closed_won FROM opportunities o WHERE o.created_date >= '2024-10-01' AND o.created_date <= '2024-12-31' AND o.stage != 'Closed Lost' GROUP BY o.last_touch_source;

The attribution window tracks all interactions 60 days before opportunity creation. The hierarchy prioritizes partner referrals, then customer success expansion, sales outbound, marketing touches, and finally product sign-ups.
